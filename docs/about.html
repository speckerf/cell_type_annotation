Project Cell Type Annotation Methods For Single Cell RNA-seq Data

Introduction

Cell-type identification is an important part of analysis of single cell RNA-seq data. Common approaches are unsupervised clustering and manual annotation based on a set of known marker genes. The downside of unsupervised clustering is that the number of clusters is usually not known in advance. The downside of manual annotation is that it is highly time-consuming and that the set of marker genes has to be identified in adance (which it often is not) (Lin et al. 2019). Thus, these approaches can be biased, because of the marker gene selection. In the past couple of years high-quality pre-labelled scRNA-seq have become more readily available, and thus a new approach has emerged: automated cell type annotation, i.e. supervised learning algorithms, which are trained on pre-labelled available reference datasets and used to automatically annotate new scRNA-seq datasets (Lin et al. 2019). The three methods that have been selected for this project are SingleR (2019), scClassify (2020) and Seurat v4 Reference Mapping (2020). SingleR and scClassify can take data in any form, whereas the Seurat Reference Mapping requires data stored as a Seurat object.

Datasets & Dataset Preprocessing

We have selected two pre-labelled pancreas scRNA-seq datasets, the Baron and Muraro dataset, which are both available from the scRNAseq library as SingleCellExperiment objects:

Baron dataset: From this paper, droplet-based scRNA-seq, 2016

Muraro dataset: From this paper, CEL-seq, 2016

The datasets will be used both as reference and as test datasets.

The datasets don’t contain the same amount of cells: the Baron dataset is larger than the Muraro dataset. But both datasets more or less contain cells of the same set of cell type labels, though we had to perform some adjustment of labels, such as renaming labels, removing unlabelled/“unknown” labels, or removing cells with labels from one dataset that was not present in the other dataset, so that eventually the two datasets contained cells from the same set of cell type labels:

  • endothelial
  • alpha: produce glucagon (endocrine)
  • beta: produce insulin (endocrine)
  • delta: produce somatostatin (endocrine)
  • epsilon: produce ghrelin (endocrine)
  • gamma: (also known as pp cells) produce pancreatic polypeptide hormone (endocrine)
  • acinar (exocrine)
  • ductal (exocrine)

Adjustment of cell type labels and quality control of the datasets were performed in the Preprocessing section.

Methods

The three methods were each run once on the Baron dataset (with Muraro as reference) and once on the Muraro dataset (with Baron as reference).

  • SingleR was run in classical mode.
  • ScClassify was run with pearson correlation as similarity metric, limma for gene selection and kNN for gene prediction, as we figured that this configuration would be most similar to SingleR. Thus, the ensemble learning was disabled.
  • The Seurat v4 Reference mapping required an additional conversion from SingleCellExperiment to a Seurat object.

The individual methods were run on the preprocessed datasets and their predictions were presented in form of cell type distributions and tSNE plots in the Methods section.

Evaluation of Methods

The predictions of the three methods were evaluated with two performance metrics:

  • adjusted Rand index
  • F1 scores for each cell type label

Discussion & Conclusion (only brainstorming)

  • reference to the scClassify paper: huge amount of combinations of datasets were compared –> how well a method performs strongly depends on the combination of datasets
  • a lot depends on the quality of the reference datasets (qc and annotation) and it requires reference dataset that contains a superset of the labels that we expect to be present in our dataset. also depends on whether you’re dealing with pancreas or PBMC scRNAseq data for example.
  • preprocessing cell type labels can be a hassle, especially if dealing with PBMC datasets, where there is a huge amount of different cell types (in high resolution)
  • personal preference whether one usually works with SingleCellExperiments (–> SingleR or scClassify) or Seurat (–> Seurat v4 Reference Mapping), performance (in our simple case) is very similar
  • this methods comparison is very “simple”
